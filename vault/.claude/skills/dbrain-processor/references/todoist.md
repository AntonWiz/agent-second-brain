# Task Integration — GTD + Milanote

> Todoist не используется. Задачи живут в двух местах:
> - **`мок/задачи-гтд.md`** — всё что нужно сделать (GTD-формат)
> - **Milanote Kanban** — проектные задачи с доской

---

## Чеклист перед созданием задачи

### 1. Проверить нагрузку (ОБЯЗАТЕЛЬНО)

Прочитать `цели/3-неделя-текущая.md` → посмотреть ежедневный план:

```
Пн: 4 задачи  ← перегружен
Вт: 2 задачи
Ср: 3 задачи  ← на пределе
Чт: 1 задача
Пт: 2 задачи
```

### 2. Проверить дубликаты (ОБЯЗАТЕЛЬНО)

Найти в `мок/задачи-гтд.md` похожую задачу.
Если есть → не создавать, пометить как дубликат.

## Приоритет по домену

| Домен | Приоритет по умолчанию | Повысить если |
|-------|----------------------|---------------|
| Проект (ГИС, Васту, Поттериана, Гудвин) | p1-p2 | дедлайн или деньги |
| YouTube | p2 | тема года |
| Команда (Алёна, Вика) | p2 | блокирует другие задачи |
| Финансы | p1-p2 | деньги зависают |
| Контент (с дедлайном) | p2-p3 | — |
| Обучение / рост | p4 | связано с суперсилой → p3 |

### Ключевые слова → приоритет

| Слова в тексте | Приоритет |
|---------------|-----------|
| срочно, горит, дедлайн, деньги | p1 |
| важно, приоритет, до конца недели | p2 |
| нужно, надо, не забыть | p3 |
| идея, потом, когда-нибудь | p4 |

### Буст через фильтры решений

Если запись совпадает с 2+ фильтрами → повысить приоритет на 1 уровень:
1. Приближает к 3М/мес?
2. Можно делегировать Алёне/Вике?
3. Отрабатывает суперсилу (извлечение сути → продукт)?
4. Это YouTube или через YouTube?

## Маппинг дат

| Контекст | Дедлайн |
|----------|---------|
| Дедлайн проекта | точная дата |
| Срочно / операционное | сегодня / завтра |
| На этой неделе | пятница |
| На следующей неделе | следующий понедельник |
| Стратегическое | +7 дней |
| Не указано | +3 дня |

### Русский текст → дата

| Фраза | Дедлайн |
|-------|---------|
| сегодня | today |
| завтра | +1 день |
| послезавтра | +2 дня |
| в понедельник | ближайший пн |
| в пятницу | ближайшая пт |
| на этой неделе | пятница |
| на следующей неделе | следующий пн |
| через неделю | +7 дней |
| 26 февраля | 2026-02-26 |

## Формат создания задачи в GTD

```markdown
- [ ] {Название задачи} | p{1-4} | {дата} | {→ Цель если есть}
```

Примеры:
```markdown
- [ ] Оплатить Кирилла Севера | p1 | 2026-02-25 | → Поттериана
- [ ] Созвониться с Викой — итоги февраля | p2 | 2026-02-27 | → ГИС
- [ ] Написать в Перезагрузку приветствие | p2 | 2026-02-24
- [ ] 1 идея для YouTube-видео | p3 | ежедневно | → YouTube
```

### Стиль названия задачи

✅ Хорошо:
- «Оплатить рекламу Кириллу Северу»
- «Созвониться с Алёной — статус видео Маркела»
- «Выложить интервью с Маркелом на YouTube»

❌ Плохо:
- «Подумать о Поттериане»
- «Что-то с YouTube»
- «Разобраться с Гудвин»

### Балансировка нагрузки

Если целевой день уже имеет 3+ задачи:
1. Найти следующий день с < 3 задачами
2. Поставить туда
3. Упомянуть в ответе: «сдвинуто на {день} — перегрузка»

## Обработка ошибок

ВАЖНО: Никогда не предлагать «добавь вручную».

Если не удалось сохранить задачу:
1. Указать точную ошибку
2. Продолжить с остальными записями
3. Не помечать как обработанное
4. Антон увидит ошибку и разберётся

НЕПРАВИЛЬНО:
> «Не удалось добавить. Добавь вручную: Оплатить Кирилла»

ПРАВИЛЬНО:
> «Ошибка при сохранении задачи: [точный текст ошибки]»

---

## Anti-Patterns (НЕ СОЗДАВАТЬ)

- ❌ «Подумать о...» → конкретизировать действие
- ❌ «Разобраться с...» → что именно сделать?
- ❌ Абстрактные задачи без следующего действия
- ❌ Дубликаты существующих задач
- ❌ Задачи без даты
